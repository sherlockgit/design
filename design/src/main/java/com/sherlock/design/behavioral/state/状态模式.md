## 状态模式
* 定义：允许一个对象在其内部状态改变时,改变它的行为
* 类型：行为型
* 优点：将不同的状态隔离；把各种状态的转换逻辑，分布到State的子类中，减少相互间依赖；增加新的状态非常简单
* 缺点：状态多的业务场景导致类数目增加，系统变复杂

### 相关设计模式
* 和享元模式：有时候可以配合使用

### 适用场景
* 一个对象存在多个状态(不同状态下行为不同)，且状态可相互转换

### 需求
* 现有一个批量操作任务，该任务流转过程中存在：待执行、执行中、已完成、执行失败几种状态，通过状态模式实现
